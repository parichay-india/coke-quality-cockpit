# 🚀 Deploy the Coke Quality Cockpit — GitHub web UI → Streamlit Cloud

A complete, **no-command-line** guide. You will create a GitHub repository in the browser, upload four files, and deploy a public dashboard on Streamlit Community Cloud. Total time ≈ 10 minutes.

---

## What you are deploying

| File | Where it comes from | Purpose |
|---|---|---|
| `app.py` | provided | the dashboard |
| `requirements.txt` | provided **(or)** generated by the notebook | Python dependencies |
| `coke_quality_model.pkl` | provided **(or)** generated by the notebook | the trained models + config |
| `.streamlit/config.toml` | provided | dark industrial theme |

> A ready-to-use `coke_quality_model.pkl` is already included, so you can deploy **right now**. If you retrain in Colab, just swap in the new `.pkl` and its `requirements.txt`.

---

## Step 0 (optional) — Regenerate the model in Colab

Skip this if you are using the provided `.pkl`.

1. Open <https://colab.research.google.com> → **File ▸ Upload notebook** → choose `CSR_CRI_Prediction_Coke_Oven.ipynb`.
2. **Runtime ▸ Change runtime type ▸ T4 GPU ▸ Save.**
3. Upload `Coke_Oven_Dataset.xls` using the 📁 file panel on the left (drag it in).
4. **Runtime ▸ Run all.** On the first run it trains automatically; on later runs it asks whether to rebuild or reuse.
5. When it finishes, the notebook downloads **`coke_quality_model.pkl`** and **`requirements.txt`**. Use *these two* files in the steps below (they guarantee the dashboard's library versions match the model).

## Step 1 — Create a free GitHub account (skip if you have one)

Go to <https://github.com/signup> and follow the prompts. A free account is enough.

---

## Step 2 — Create a new repository

1. Click the **+** at the top-right of GitHub → **New repository**.
2. **Repository name:** `coke-quality-cockpit` (any name is fine).
3. Set it to **Public** (Streamlit's free tier needs a public repo).
4. Tick **Add a README file**.
5. Click **Create repository**.

---

## Step 3 — Upload the app files

You will do this twice because `.streamlit/config.toml` lives in a sub-folder.

### 3a · Upload the three top-level files

1. On the repo page, click **Add file ▸ Upload files**.
2. Drag in **`app.py`**, **`requirements.txt`**, and **`coke_quality_model.pkl`** together.
   - The `.pkl` is ~24 MB — well under GitHub's 25 MB web-upload and 100 MB file limits, so no Git LFS is needed.
3. In **Commit changes**, leave the default message and click **Commit changes**.

### 3b · Add the theme file in a `.streamlit` folder

1. Click **Add file ▸ Create new file**.
2. In the filename box type exactly:
   ```
   .streamlit/config.toml
   ```
   Typing the `/` automatically creates the folder.
3. Open the provided `.streamlit/config.toml`, copy its contents, and paste them into the editor.
4. Click **Commit changes**.

Your repo root should now contain: `app.py`, `requirements.txt`, `coke_quality_model.pkl`, `README.md`, and a `.streamlit/` folder.

---

## Step 4 — Deploy on Streamlit Community Cloud

1. Go to <https://share.streamlit.io> and click **Sign in with GitHub** (authorise access when asked).
2. Click **Create app** ▸ **Deploy a public app from GitHub**.
3. Fill in:
   - **Repository:** `your-username/coke-quality-cockpit`
   - **Branch:** `main`
   - **Main file path:** `app.py`
4. Click **Deploy**.

The first build takes 2–4 minutes while it installs the libraries and unpickles the model. When it finishes you get a public URL like
`https://coke-quality-cockpit.streamlit.app` — share it with anyone.

> **Updating later:** edit or re-upload any file on GitHub (e.g. drop in a freshly trained `.pkl`). Streamlit Cloud auto-redeploys on every commit. You can also click **Reboot** from the app's **⋮** menu.

## Troubleshooting

**"coke_quality_model.pkl not found."**
The `.pkl` must sit in the repo **root**, next to `app.py` (not inside a folder). Re-upload it there.

**Unpickling error / `InconsistentVersionWarning` / "cannot load model".**
The dashboard's library versions don't match the ones that built the `.pkl`. Fix it by using the **`requirements.txt` the notebook generated** (Step 0), which pins the exact `scikit-learn`, `xgboost`, `lightgbm`, and `catboost` versions. Replace `requirements.txt` in the repo with that file and reboot the app. *(The provided `.pkl` already matches the provided `requirements.txt`.)*

**Build fails on `catboost` or `lightgbm`.**
If you trained with only some libraries, you can delete the unused lines from `requirements.txt` — the dashboard only needs the library of the **winning** model (shown on the dashboard's model card) plus `streamlit`, `pandas`, `numpy`, `scikit-learn`, `plotly`, and `joblib`.

**App is asleep ("Yes, get this app back up").**
Free-tier apps sleep after inactivity. Anyone can click the button to wake it in a few seconds.

**Upload is too large.**
Only if you exceed 25 MB per file in the browser: upload that file via **Add file ▸ Upload files** one at a time, or retrain with fewer trees to shrink the `.pkl`. The current model is fine.
